##  Ответы на вопросы

### 1. Что такое непрерывная интеграция?
Непрерывная интеграция (CI) — это практика разработки, при которой изменения кода автоматически собираются и тестируются при каждом обновлении репозитория.

---

### 2. Для чего нужны юнит-тесты? Как часто их запускать?
Юнит-тесты проверяют корректность работы отдельных частей программы.  
Их нужно запускать:
- при каждом изменении кода
- автоматически через CI

---

### 3. Как запускать тесты при Pull Request?

Нужно изменить блок `on`:

```yaml
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
```

---

### 4. Как удалять Docker-образы после тестов?

Добавить шаг:

```yaml
- name: Remove Docker image
  run: docker rmi containers08
```
